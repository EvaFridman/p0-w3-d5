## Контекст

Транслитерация — это запись русского текста латинскими буквами: `собака сутулая` → `sobaka
sutulaya`. Она нужна везде, где кириллица не проходит: в загранпаспортах, url-адресах, никах,
именах файлов. Ты делаешь простой транслитератор: пользователь вводит русский текст, нажимает
кнопку — и под формой появляются два столбца: слева оригинал, справа его латинская запись. Это
сводит воедино весь пройденный материал: верстку, адаптив и CSS (недели 1–2), функции, условия и
регулярные выражения (неделя 3), работу с DOM (последний день).

```
русский текст в инпут → транслитерация в латиницу
┌──────────────────────────────────────────────────────┐
│ ввод: 'собака сутулая'                               │
│         │  валидация: не пусто, нет мата             │
│         │  таблица замен (щ→shch раньше с→s)         │
│         ▼                                            │
│  оригинал        │        транслит                   │
│  собака сутулая  │        sobaka sutulaya            │
└──────────────────────────────────────────────────────┘
верстка+CSS (нед.1-2) · функции/регулярки (нед.3) · DOM
```

## ТЗ

Одностраничное приложение на чистом HTML/CSS/JS. Пользователь вводит русский текст в поле,
нажимает кнопку «Транслитерировать» — приложение валидирует ввод и выводит под формой результат в
два столбца: слева исходный текст, справа — транслитерация латиницей. Никакой сети и сторонних
библиотек: вся логика — своя.

Что приложение должно делать (минимум):

- **Транслитерация.** Заменить каждую русскую букву на латинское соответствие по таблице
  (`а→a`, `б→b`, … `я→ya`). Сохранить регистр (`Собака` → `Sobaka`) и не-буквенные символы
  (пробелы, цифры, знаки) как есть.
- **Многобуквенные сочетания — обязательно учесть.** Часть букв даёт несколько латинских
  (`ж→zh`, `ч→ch`, `ш→sh`, `щ→shch`, `ё→yo`, `ю→yu`, `я→ya`, `ц→ts`). Их нужно обработать
  РАНЬШЕ одиночных букв, иначе результат склеится неверно. Продумай порядок замен — это главная
  ловушка задачи.
- **Валидация ввода.** Пустой текст или текст из одних пробелов не обрабатывается: под полем
  показывается ошибка, рамка поля подсвечивается красным, вывод не строится.
- **Фильтр мата.** Если во вводе есть нецензурная лексика — не транслитерировать, показать под
  полем ошибку и подсветить рамку красным. Проверка делается по своему списку запрещённых основ
  (массив в JS): привести ввод к нижнему регистру и проверить вхождение основ из списка. Учти
  честно: простой список не поймает намеренные обходы (латиницей, через звёздочки, по буквам) —
  это нормально для учебного фильтра; задача не «отловить всё», а сделать работающую базовую
  проверку и понимать её границы (отметить это в `NOTES.md`).
- **Стартовое состояние с примером.** При первом открытии страница не пустая: в поле уже
  подставлен пример (например `собака сутулая`), и под формой сразу показан его результат в два
  столбца — чтобы было видно, как приложение работает, ещё до первого ввода.
- **Удаление строк.** Если ввод многострочный, каждая строка выводится отдельной парой
  «оригинал | транслит» со своей кнопкой удаления (`✕`); нажатие убирает эту пару из результата
  (`remove`). Удобно обработать клики делегированием — один обработчик на контейнере результата.
- **Вывод в DOM.** Результат в два столбца строится скриптом (`createElement`/`textContent`) в
  пустой контейнер. Текст пользователя выводится через `textContent`, не `innerHTML` (XSS с
  прошлого дня).

Требования к качеству (планка недели):

- Логика разнесена по функциям (валидатор, проверка мата, транслитератор, вывод), а не одним
  куском; функции переиспользуемы и вызываются по имени.
- Таблицу замен держать данными (объект), а не лесенкой из `if`. Порядок обработки многобуквенных
  сочетаний осознанный, ты можешь его объяснить.
- Верстка семантическая и доступная (форма с `label`, кнопка, область результата), адаптивная
  на трёх ширинах — мобильный, планшет, десктоп (mobile-first, `@media` из недели 2): на узком
  экране столбцы «оригинал | транслит» перестраиваются так, чтобы не было горизонтальной
  прокрутки. Аккуратный CSS — переменные, состояния фокуса и **наведения (`:hover`)**: кнопки
  и крестик удаления заметно реагируют на курсор (крестик активной строки на `:hover` —
  красный), красная рамка ошибки.
- Ввод и результат безопасны: текст пользователя не попадает в разметку как HTML.
- Ни одной сторонней библиотеки и обращений к сети.

## Критерии приёмки

* При первом открытии поле уже содержит пример и его результат показан под формой.
* Приложение принимает русский текст, валидирует его и выводит два столбца (оригинал | транслит)
  в DOM без перезагрузки.
* Многострочный ввод выводится построчно, каждую строку результата можно удалить кнопкой `✕`.
* Транслитерация верная: одиночные и многобуквенные сочетания (`ж`, `ч`, `ш`, `щ`, `ё`, `ю`,
  `я`, `ц`) обработаны в правильном порядке, регистр сохранён, пунктуация и цифры не потеряны.
* Пустой/пробельный ввод и ввод с матом отклоняются: под полем ошибка, рамка поля красная,
  результат не строится.
* Логика разнесена по функциям; таблица замен — данными, а не цепочкой `if`.
* Интерактивные элементы имеют состояния `:hover` и `:focus-visible` (кнопки, крестик удаления); фокус виден с клавиатуры.
* Верстка семантическая, доступная и адаптивная на мобильном, планшете и десктопе (без
  горизонтальной прокрутки на узком экране); текст выводится безопасно (`textContent`).
* Нет сторонних библиотек и сети. В репозитории есть `RELEASES.md` (план записан до кода) и
  `NOTES.md` (границы фильтра мата).
